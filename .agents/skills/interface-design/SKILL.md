## What I do

- Review existing UI components and identify design inconsistencies or accessibility gaps
- Propose layout structures using established patterns (grid, flexbox, stacking)
- Implement responsive designs that work across mobile, tablet, and desktop breakpoints
- Apply accessible markup: semantic HTML, ARIA labels, keyboard navigation, contrast ratios
- Enforce visual consistency: spacing scales, typography hierarchy, color tokens
- Suggest and apply component decomposition to keep UI code modular and reusable
- Review designs against WCAG 2.1 AA standards

## When to use me

Use this skill when:
- Building or refactoring a screen, page, or component
- You need guidance on layout, spacing, or visual hierarchy
- Accessibility compliance is required
- You want to ensure cross-platform or cross-browser consistency
- You are designing a new UI feature from scratch

## How I work

1. Understand the target platform and any existing design system or theme
2. Identify the user interaction model (touch vs. pointer, screen reader support)
3. Propose a component structure before writing code
4. Implement with semantic markup and scoped styles
5. Verify responsiveness and accessibility before finishing
